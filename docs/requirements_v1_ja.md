# Amazon Product Discovery Engine 要件定義書 v1.0

## 1. 目的

AmazonにおけるFBA/OEM向け商品を、カテゴリ指定だけで自動発掘し、参入可否を判断するツールを構築する。

## 2. 利用者

- 自社メンバー（物販担当）
- 非エンジニアでも利用できるUI/運用を前提とする

## 3. ユースケース

### UC-01: 商品探索

- カテゴリを入力する
- AIが商品候補を提示する

### UC-02: 商品分析

- ASIN単位で分析する
- GO / NO-GO判定を返す

### UC-03: 比較検討

- 複数商品を一覧で比較する

### UC-04: 深掘り

- LLMで戦略・差別化案を取得する

## 4. 機能要件

### 4.1 商品探索機能

**入力**

- カテゴリ（例: デスク周り）
- 価格帯（任意）
- レビュー数条件（任意）

**処理**

- LLMで検索キーワードを生成
- Amazon検索でASIN収集
- Keepa APIでデータ取得

**出力**

- 商品リスト（10〜100件）

### 4.2 スコアリング機能（コア）

各ASINに対してスコア算出を行う。

#### 評価軸

- 売上安定性（0-30）
  - BSRの変動
  - トレンド
- 価格健全性（0-25）
  - 価格下落率
  - セール頻度
- 競争強度（0-25）
  - 出品者数
  - Buy Box分散
- 参入余地（0-20）
  - レビュー数
  - ブランド強度

**出力**

- 総合スコア（0-100）
- 判定基準
  - 80以上: GO
  - 60-79: 条件付きGO
  - 59以下: NO-GO

### 4.3 商品一覧表示

**表示項目**

- 商品名
- スコア
- 想定月商（簡易推定）
- 競争レベル
- 判定

### 4.4 商品詳細画面

**セクション**

1. 結論
   - GO / NO-GO
   - 理由（3点）
2. リスク
   - 主要リスク2〜3点
3. データ
   - Keepaグラフ
   - 出品者推移
4. アクション
   - 「OEM検討」
   - 「類似商品」

### 4.5 LLM分析機能

**使用モデル**

- Google Gemini（無料枠）

**機能**

- 分析レポート生成
- 差別化提案
- Q&A（チャット）

### 4.6 キャッシュ機能

- 同一ASINは再分析しない
- 有効期限: 24時間

### 4.7 バッチ処理

- 定期的に人気商品を更新
- Cron実行

## 5. 非機能要件

### 5.1 パフォーマンス

- 商品一覧表示: 3秒以内
- 詳細分析: 5秒以内

### 5.2 可用性

- 個人利用のため高可用性は不要
- エラー時はリトライ

### 5.3 セキュリティ

- Supabase Authで認証
- APIキーはサーバー側管理

### 5.4 コスト制約

- 月額1万円以内

## 6. システム構成

- フロント: Next.js（Vercel）
- バックエンド: Vercel Functions（API）
- DB: Supabase（PostgreSQL）
- 外部API: Keepa API, Gemini API

## 7. データモデル

### products

- asin
- title
- category

### keepa_data

- asin
- price_history (jsonb)
- bsr_history (jsonb)
- updated_at

### analysis

- asin
- score
- decision
- summary
- created_at

## 8. 処理フロー

### 探索フロー

カテゴリ入力 → キーワード生成（LLM）→ ASIN取得 → Keepa取得 → スコアリング → 一覧表示

### 詳細分析

ASIN選択 → DB参照 → スコア算出 → LLM分析 → 表示
